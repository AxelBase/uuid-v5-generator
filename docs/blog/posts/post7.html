Why This Tool Exists (And Why It’s 100% Client-Side)

Published: December 4, 2025

I built this UUID v5 generator for one simple reason: every other tool was wrong.

Most online generators only offer v4. Some claim v5 but use flawed implementations. Almost all send your data to servers. None work offline.

This one fixes all of that.

What Makes This Different

  • 100% RFC 4122 compliant (verified with official test vectors)
  • Runs entirely in your browser — zero network requests after load
  • Full offline support via PWA
  • No cookies, no analytics, no fingerprinting
  • Bulk generation up to 10,000 IDs
  • Open source, auditable, MIT licensed

Built for Real Engineers

This isn’t a toy. It’s the tool I wish existed when:

  • Debugging distributed tracing
  • Seeding test databases
  • Building offline-first prototypes
  • Working in secure environments

The Future

Planned features:

  • UUID v7 support (when stable)
  • Namespace manager
  • CLI version
  • Browser extension
