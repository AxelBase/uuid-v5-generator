The Four Official Namespaces Explained

The Four Official Namespaces Explained

Published: November 30, 2025

When RFC 4122 defined UUID v5, it also specified four canonical namespace UUIDs. These are sacred — hard-coded values that ensure global consistency when generating name-based IDs from common domains.

1. DNS Namespace

UUID: 6ba7b810-9dad-11d1-80b4-00c04fd430c8

Use when your name is a domain name.

uuid_v5(DNS, "github.com") →
cfa3f64d-5d7f-5d0f-8a7b-9e8d8f8c8d9e

2. URL Namespace

UUID: 6ba7b811-9dad-11d1-80b4-00c04fd430c8

For full URLs — including protocol and path.

uuid_v5(URL, "https://axelbase.dev/tools/uuid-v5") → same everywhere

3. OID Namespace

UUID: 6ba7b812-9dad-11d1-80b4-00c04fd430c8

For ISO Object Identifiers (used in SNMP, LDAP, etc.)

4. X.500 Namespace

UUID: 6ba7b814-9dad-11d1-80b4-00c04fd430c8

For X.500 Distinguished Names (DNs) in directories.

Why This Matters

Using these official namespaces means:

  • Your generated IDs are interoperable with other systems
  • Tools can recognize the input type
  • You follow the standard exactly

Pro Tip

Always store both the name and namespace when using v5 — this allows reconstruction and debugging later.

FAQ

Can I create my own namespace?

Yes! Generate a random UUID v4 and use it as your private namespace. Just document it.

Are these namespaces "magic"?

No — they’re just well-known v4 UUIDs reserved by the RFC. Anyone can verify them.

The four namespaces are the foundation of interoperable, deterministic identifiers. Master them — master UUID v5.
